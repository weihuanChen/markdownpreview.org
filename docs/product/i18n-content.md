## DEFAULT_MARKDOWN 编辑器模板

编辑器模板文件位置：`public/templates/`

### 文件列表

- `default-ja.md` - 日语 Markdown 教程模板
- `default-en.md` - 英语 Markdown 教程模板
- `default-zh.md` - 中文 Markdown 教程模板

### 模板结构

所有语言的模板都包含以下部分：

1. **欢迎标题** - 项目名称和欢迎语
2. **标题示例** - H1-H4 标题的用法
3. **文本样式** - 粗体、斜体、删除线
4. **列表** - 无序列表、有序列表、任务列表
5. **引用** - Blockquote 示例
6. **代码** - 内联代码和代码块（JavaScript 示例）
7. **数学公式** - LaTeX 内联和块级公式
8. **表格** - Markdown 表格示例
9. **链接** - 外部链接示例
10. **脚注** - 脚注引用和定义

### 内容本地化原则

- **完全翻译**：所有说明文字、注释、示例内容都翻译成对应语言
- **保持一致**：代码示例的功能名称和注释保持一致（如 `fibonacci` 函数）
- **文化适配**：表格中的人名和示例数据适配目标语言文化

## 文案更新流程

### 修改现有文案

1. 定位文案键：在 `messages/*.json` 文件中找到对应的翻译键
2. 同步修改：同时修改三个语言文件（ja.json, en.json, zh.json）
3. 验证一致性：确保所有语言的文案键都存在且含义一致
4. 测试显示：在应用中切换语言，验证文案显示正确

### 添加新文案

1. 确定翻译键命名（遵循现有命名规范）
2. 在三个 JSON 文件中同时添加新键值对
3. 在组件中使用 `t('new_key')` 引用新文案
4. 更新本文档，记录新文案内容

### 文案审核清单

- [ ] 所有三种语言的文案都已提供
- [ ] 文案长度适中，不会导致 UI 错乱
- [ ] 文案语气、风格与现有内容一致
- [ ] 专业术语翻译准确（如 Markdown、LaTeX 等）
- [ ] 标点符号使用正确（日语全角、中文全角、英语半角）

## 文案风格指南

### 日语 (ja)

- 使用礼貌体（です・ます体）
- 技术术语优先使用片假名（カタカナ）
- 适当使用汉字，避免过度使用

### 英语 (en)

- 使用简洁、专业的技术写作风格
- 避免过于口语化的表达
- 使用主动语态

### 中文 (zh)

- 使用简体中文
- 技术术语保留英文或使用约定俗成的中文翻译
- 语句简洁明了，避免冗长
