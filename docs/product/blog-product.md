# Blog 模块功能需求文档

- **版本:** 1.0
- **日期:** 2025-11-15
- **负责人:** [markdownpreview.org]

## 1. 概述

本文档定义了 Blog 模块的前端功能需求，重点关注列表页的实现和详情页的渲染方案。

**范围说明:**

- **包含:** 列表页 UI/UX、详情页 UI/UX、Markdown 解析与渲染。
- **不包含:** 后端 API 设计、数据库表结构、数据接入方案 (已由您方准备)。

## 2. 详细功能需求

### 2.1 Blog 列表页

**目标:** 以清晰、信息丰富的方式展示所有博文的概览，并支持用户浏览。

**功能点:**

- **Blog 卡片 (Blog Card) 布局**
  - 列表页应采用卡片（Card）布局，每个卡片代表一篇博文。
  - 每个卡片必须 **必须** 包含以下四个核心元素：
    1. `**标题 (Title):**` 博文的主标题。
    2. `**作者 (Author):**` 博文的作者名。
    3. `**摘要 (Summary):**` 博文的简短摘要或描述。
    4. `**标签 (Tags):**` 该博文关联的标签列表 (应支持多个标签的展示)。
- **分页 (Pagination)**
  - 列表页底部 **必须** 实现一个分页组件。
  - 组件应允许用户通过点击页码（例如：1, 2, 3...）或“上一页”/“下一页”按钮来导航。
  - 应向用户清晰展示当前页码和总页数（或是否有更多页面）。

### 2.2 Blog 详情页

**目标:** 高效渲染单篇博文内容，并提供良好的阅读辅助功能（如目录）。

**功能点:**

- **Markdown 渲染方案**

  - 博文正文（Markdown 内容）的渲染 **必须** 采用 https://streamdown.ai/ 方案来实现。

- **摘要 (Summary) 渲染**

  - 在博文正文的顶部（或正文开始前）， **必须** 展示该博文的摘要。

  - 此摘要 **必须** 统一使用 **Markdown 的引用（Blockquote）** 样式进行渲染。

  - **渲染示例:**

    > 这是博文的摘要内容。它将以这种灰底或带有竖线的引用样式显示在页面上，与正文内容作出区分。

- **目录 (TOC - Table of Contents) 组件**

  - 页面 **必须** 提供一个 TOC (目录) 组件（通常显示在页面侧边栏）。
  - 该组件 **必须** 能够 **自动解析** 当前博文 Markdown 内容中的所有标题（例如 `<h1>`, `<h2>`, `<h3>` 等）。
  - 目录应正确反映标题的层级结构（例如，`<h3>` 标题应缩进显示在对应的 `<h2>` 标题下方）。
  - 点击 TOC 中的任意一个标题链接，页面 **必须** 平滑滚动到正文中对应的标题位置。
